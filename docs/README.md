# TranslationStack Docs

This directory contains the public documentation for TranslationStack.

## Start Here

- [Design](design/FINAL_DESIGN.md) - product position, protocol model, project files, lifecycle, and validation rules
- [DOCX and Competitor Research](research/DOCX-AND-COMPETITORS.zh.md) - adapter strategy and product lessons from adjacent tools
- [Implementation Notes](archive/BUILDING_TRANSLATESTACK.md) - practical build notes for contributors
- [Examples](../examples/) - complete demo projects that can be inspected or validated locally

## Reading Order

1. Read the repository [README](../README.md) for the public overview.
2. Read the [Design](design/FINAL_DESIGN.md) to understand the contract-first architecture.
3. Read the [Research Notes](research/DOCX-AND-COMPETITORS.zh.md) before adding format adapters or copying competitor behavior.
4. Inspect the [Building Effective Agents demo](../examples/building-effective-agents/) to see the protocol files in a complete project.
5. Read the [Implementation Notes](archive/BUILDING_TRANSLATESTACK.md) before changing validation, state, persistence, or review/export behavior.

## Documentation Principles

- Public docs describe durable product decisions, not private experiments.
- Implementation claims should be easy to verify from files in this repository.
- Time-sensitive package versions, star counts, benchmark numbers, and local machine paths should not be treated as stable documentation.
- The project should be presented as a professional translation workflow protocol, not as a prompt collection or one-off demo.
